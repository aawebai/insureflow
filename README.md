# InsureFlow 智保通 —— AI 原生保险智能中台

[![Dify](https://img.shields.io/badge/Built%20with-Dify-1E90FF)](https://dify.ai)
[![Python](https://img.shields.io/badge/Python-3.10+-3776AB)](https://www.python.org/)

**让保险业务人员用自然语言完成「产品配置 → 核保决策 → 保单服务」全链路操作。**

对标 eBaoTech InsureMO，用 Dify + RAG + Multi-Agent 搭建的 AI 保险中台 Demo。从需求分析、系统设计、Dify 实操踩坑到测试验证全程独立完成。

## 核心亮点

### 模块B：多 Agent 协同核保引擎（核心模块）

5 个专业化 Agent 分别负责：信息提取 → 职业风险评估 → 健康风险评估 → 多维融合定价 → 报告生成。关键路径并行执行，减少约 50% 延迟。

### RAG 知识库 + 结构化查询构造器

覆盖 6 级职业风险、8 类健康状况、4 种险种的核保规则知识库。解决了「自然语言输入与结构化规则文本语义鸿沟」问题——在 LLM 提取和知识库检索之间加入结构化查询构造层，检索召回率从几乎为 0 提升至稳定可用。

### LLM 输出容错体系

所有 Code 节点实现统一 `safe_parse()` 函数，清洗 `<think>` 标签、提取 JSON 子串、降级兜底。配合 LLM 节点重试机制（最大3次）+ Assistant Prefill，确保系统在 LLM 输出不确定的情况下不崩溃。

### API 级联架构（双方案）

- **Scheme A（Hub）**：统一智能中枢，嵌入式 Prompt 路由，Dify Cloud 可运行，用于日常演示
- **Scheme B（API 编排）**：Python 微服务编排脚本，调用各模块 Workflow API 实现全链路串联。排查过程中使用三级控制变量法定位 Dify Cloud 沙箱限制，体现工程化调试能力

## 关键技术决策

| 决策点 | 选项 | 选择 | 理由 |
|--------|------|------|------|
| 搭建平台 | 纯代码 vs Dify | Dify | 快速验证想法，可视化编排降低迭代成本 |
| 架构模式 | 单 LLM vs 多 Agent | 多 Agent | 核保多维度评估，并行减少延迟，单一Agent专注度更高 |
| 知识管理 | 硬编码 vs RAG | RAG | 保险规则频繁变化，RAG支持热更新，改文档即生效 |
| 执行方式 | 串行 vs 并行 | 关键路径并行 | Agent2和Agent3独立评估不同维度，并行可减少约50%延迟 |
| API级联 | Hub vs 微服务编排 | 双方案 | Cloud限制下Hub用于演示，Python脚本展示架构能力 |
