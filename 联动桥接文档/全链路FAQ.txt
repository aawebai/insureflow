# 全链路 FAQ — 跨模块常见问题与处理策略

## 用途
用户在全链路（产品→核保→保单）过程中可能提出的跨模块问题及系统应如何响应。供 Hub 检索使用。

---

## Q1: 核保时被拒保了，能换一款产品吗？
**跨模块**：B → A
**处理策略**：
1. 从 underwriting_context 中提取拒保原因
2. 自动路由到模块A，根据用户画像匹配核保更宽松的替代产品
3. 匹配逻辑：拒保重疾险 → 推荐防癌险/惠民保；拒保医疗险 → 推荐意外险+特定疾病险
4. 同时检查模块A知识库中是否有适配该用户情况的产品模板

## Q2: 核保通过后，保单什么时候生效？
**跨模块**：B → C
**处理策略**：
1. 从 product_context 读取 waiting_period_days
2. 从 underwriting_context 读取核保决策时间
3. 计算生效日期：核保通过次日零时
4. 计算等待期结束日期
5. 接入模块C的保单解读能力解释生效和等待期的区别

## Q3: 这个产品的某条免责条款我不太理解？
**跨模块**：B → A / C
**处理策略**：
1. 从 product_context 中定位具体产品的 exclusions 字段
2. 从统一知识库检索「常见拒赔案例与解读」
3. 用大白话解释每条免责条款 + 配一个真实案例说明
4. 强调"这是行业通用条款"降低用户顾虑

## Q4: 我已有保单，想再加保额？
**跨模块**：C → B
**处理策略**：
1. 先走模块C查现有保单状态
2. 确认保单仍在有效期内
3. 进入模块B做加保核保：只需要核保新增保额部分
4. 加保核保采用更简化的流程（已知用户基本信息）
5. 汇总展示：原保单 + 新加保部分的完整保障

## Q5: 我在两家公司都买了医疗险，能重复理赔吗？
**跨模块**：C（理赔咨询）
**处理策略**：
1. 检索知识库「费用补偿原则」
2. 解释：医疗险是实报实销，总赔付不超过实际花费
3. 建议：保留保障更好的一家的医疗险，另一家可考虑转投重疾险
4. 禁止建议退保（合规要求）

## Q6: 我怀孕了/换工作了/搬去外地了，保单怎么办？
**跨模块**：C（保单变更服务）
**处理策略**：
1. 检索「续保与保单服务规则」中的保单变更章节
2. 换工作：重点关注职业等级变化 → 若等级上升需通知保险公司
3. 搬去外地：确认保险公司在目的地是否有分支机构
4. 怀孕：告知生育相关保障的等待期和限制

## Q7: 我这个月忘记缴费了，保单还有效吗？
**跨模块**：C（续保规则）
**处理策略**：
1. 检索 product_context 中的宽限期设置
2. 判断当前日期是否在宽限期内（通常60天）
3. 宽限期内：保障有效，尽快缴费即可
4. 超过宽限期：引导走保单复效流程
5. 超过2年：说明保单已永久失效

## Q8: 我想给我爸妈也买一份？
**跨模块**：A → B
**处理策略**：
1. 分析爸妈年龄是否在产品定义的 target_age_range 内
2. 若超过年龄上限：推荐老年专属产品
3. 若在范围内：走模块B核保（注意高龄核保更严格）
4. 保险利益确认：子女对父母有保险利益关系

## Q9: 我体检发现XXX，需要告诉保险公司吗？
**跨模块**：C → B
**处理策略**：
1. 分两种情况：
   - 保单生效前（核保阶段）：必须告知，否则构成未如实告知
   - 保单生效后（等待期后）：一般不需要，但理赔时会核查
2. 如体检异常可能影响核保结果，建议告知
3. 检索「常见拒赔案例」中未如实告知导致的拒赔案例作为警示

## Q10: 我买的这个产品和XX公司的比哪个好？
**跨模块**：C → A
**处理策略**：
1. 从 product_context 获取当前产品参数
2. 从统一知识库检索「市场热门产品参数表」中的竞品数据
3. 做对比分析：保额/费率/等待期/特色条款/免责范围
4. 客观呈现差异，不做"更好/更差"的绝对判断
5. 标注"以上对比仅基于公开信息，具体以条款为准"

```
用户提出问题
    │
    ▼
LLM 意图识别 + 关键词匹配
    │
    ├── 命中了 FAQ 中的某条 → 按处理策略执行
    ├── 问题跨模块但 FAQ 未覆盖 → 检索统一知识库 + 判断主要模块

```
