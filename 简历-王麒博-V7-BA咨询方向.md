## 调整逻辑

### 为什么求职意向不写具体岗位？

内推 + HR分配部门 + 两个相关岗位（BA/咨询）JD完全一样 = 写死任何一个岗位都是自我设限。让个人简介和项目经历自然把你定位在"懂保险业务 + 能做需求分析 + 英语好"这个交叉点上，HR一看就知道往哪放。

### 为什么项目经历换了一个讲法？

核保助手这个项目本身没变，但叙述角度从"我做了个AI产品"变成了"我分析了一个保险业务场景，拆解了需求，写了文档"。这是BA和PM叙事的核心区别：

| PM叙事（V6） | BA叙事（V7） |
|-------------|-------------|
| 我搭建了核保工作流 | 我分析了核保业务流程，拆解为5个环节 |
| 我优化了检索召回率 | 我发现用户表达和系统数据之间存在落差 |
| 我设计了容错机制 | 我输出了PRD和测试文档 |
| 我总结了三个踩坑经验 | 我理解了需求分析中"翻译层"的价值 |

### 为什么英语提到前面？

两个JD的任职要求第一条都是"英语听说读写卓越"。你的CET-6放在"证书"角落里太低调了，提到个人信息区 + 个人简介 + 技能第一条。
